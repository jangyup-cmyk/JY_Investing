 **현재 구조를 유지하는 전제**로, 바로 구현에 들어갈 수 있는 **STEP 1 파일별 구현 초안**

README 기준 현재 프로젝트는 이미 `src/infra`, `src/web`, `src/workers` 중심 구조를 갖고 있고, API도 `/api/telegram/channels`, `/sync`, `/toggle`가 정의되어 있으며, Phase 2 확장 중심도 `src/workers`와 `src/infra/telegram`으로 잡혀 있습니다. 그래서 이번 초안은 **폴더 대공사 없이 현재 구조에 필요한 파일만 추가/보강**하는 방식이 가장 자연스럽습니다.

# STEP 1 파일별 구현 초안

## 0. 이번 단계의 목표

이번 STEP 1의 산출물은 아래 3개입니다.

* **표준 데이터 계약**
* **공통 메시지 저장 경로**
* **채널 상태 변경 API**

즉, 아직 실시간 리스너 전체를 완성하는 단계가 아니라,
**백필과 실시간 수집이 공통으로 사용할 저장 뼈대**를 먼저 만드는 단계입니다. README상 Phase 2 목표가 “실시간 이벤트 리스너, 채널 동기화, 메시지 DB 적재”인 만큼, 지금 단계는 그중 **동기화와 적재 규약**을 먼저 닫는 작업으로 보면 됩니다.

## 1. `src/infra/db/schemas.py`  **[신규 추가]**

### 역할

DB 모델과는 별개로, **저장/응답/상태 전이용 타입 계약**을 모아두는 파일입니다.

### 여기 넣을 것

* `ChannelSyncStatus`
* `ChannelCollectionStatus`
* `UpsertAction`
* `TelegramMessagePayload`
* `UpsertMessageResult`
* `ToggleChannelRequest`
* `ToggleChannelResponse`
* `SyncChannelsResponse`

### 추천 예시

```python
from dataclasses import dataclass
from datetime import datetime
from enum import Enum
from typing import Optional


class ChannelSyncStatus(str, Enum):
    DISCOVERED = "DISCOVERED"
    SYNCED = "SYNCED"
    MISSING = "MISSING"
    INACCESSIBLE = "INACCESSIBLE"


class ChannelCollectionStatus(str, Enum):
    PENDING_BACKFILL = "PENDING_BACKFILL"
    ACTIVE = "ACTIVE"
    PAUSED = "PAUSED"
    ERROR = "ERROR"


class UpsertAction(str, Enum):
    CREATED = "created"
    UPDATED = "updated"
    UNCHANGED = "unchanged"
    FAILED = "failed"


@dataclass
class TelegramMessagePayload:
    telegram_channel_id: int
    tg_message_id: int
    sender_id: Optional[int]
    message_text_raw: Optional[str]
    reply_to_message_id: Optional[int]
    views: Optional[int]
    forwards: Optional[int]
    grouped_id: Optional[int]
    edited_at: Optional[datetime]
    is_deleted: bool
    message_date: datetime
    collected_at: datetime
    raw_payload_json: str


@dataclass
class UpsertMessageResult:
    action: UpsertAction
    telegram_channel_id: int
    tg_message_id: int
    checkpoint_candidate: Optional[int]
    error_message: Optional[str] = None
```

### 포인트

이 파일의 목적은 “비즈니스 로직”이 아니라 **타입 표준화**입니다.
지금 단계에서는 여기에 Enum과 Result 타입을 모아두는 것이 가장 실용적입니다.

## 2. `src/infra/db/repository.py`  **[신규 추가]**

### 역할

실제 SQLAlchemy를 사용하는 **영속화 로직 전담 파일**입니다.
즉, `upsert_message()`와 체크포인트 갱신을 여기로 모읍니다.

### 핵심 함수

* `get_or_create_channel_state(...)`
* `upsert_message(db, payload)`
* `update_channel_checkpoint(db, telegram_channel_id, candidate_message_id, collected_at)`
* `mark_channel_sync_status(...)`
* `set_channel_collection_status(...)`
* `list_channels_with_state(...)`

### 최소 구현 함수 초안

```python
async def upsert_message(db: AsyncSession, payload: TelegramMessagePayload) -> UpsertMessageResult:
    ...

async def update_channel_checkpoint(
    db: AsyncSession,
    telegram_channel_id: int,
    candidate_message_id: int,
    collected_at: datetime,
) -> None:
    ...

async def set_channel_collection_status(
    db: AsyncSession,
    channel_id: int,
    new_status: ChannelCollectionStatus,
    last_error_message: str | None = None,
) -> None:
    ...
```

### 핵심 규칙

`upsert_message()` 안에서 반드시 고정할 규칙은 아래입니다.

* 유니크 기준: `(telegram_channel_id, tg_message_id)`
* 신규면 `CREATED`
* 기존 메시지에 변경이 있으면 `UPDATED`
* 완전히 동일하면 `UNCHANGED`
* 예외면 rollback 후 `FAILED`
* 체크포인트는 이 함수 안에서 직접 올리지 말고, **호출부에서 성공 결과일 때만** `update_channel_checkpoint()`를 따로 호출

### 왜 분리하나

이렇게 해두면

* 백필 루프
* 실시간 이벤트 핸들러
  둘 다 같은 저장 함수를 쓸 수 있습니다.

이게 STEP 1의 핵심입니다.

## 3. `src/infra/telegram/translators.py`  **[신규 추가]**

### 역할

Telethon 객체를 내부 표준 payload로 바꾸는 **번역기**입니다.

### 핵심 함수

* `serialize_telethon_message(message) -> str`
* `message_to_payload(message, telegram_channel_id) -> TelegramMessagePayload`
* 필요 시 `edited_event_to_payload(...)`

### 구현 초안

```python
from datetime import datetime, timezone

def message_to_payload(message, telegram_channel_id: int) -> TelegramMessagePayload:
    return TelegramMessagePayload(
        telegram_channel_id=telegram_channel_id,
        tg_message_id=message.id,
        sender_id=getattr(message, "sender_id", None),
        message_text_raw=getattr(message, "message", None),
        reply_to_message_id=getattr(getattr(message, "reply_to", None), "reply_to_msg_id", None),
        views=getattr(message, "views", None),
        forwards=getattr(message, "forwards", None),
        grouped_id=getattr(message, "grouped_id", None),
        edited_at=getattr(message, "edit_date", None),
        is_deleted=False,
        message_date=message.date,
        collected_at=datetime.now(timezone.utc),
        raw_payload_json=serialize_telethon_message(message),
    )
```

### 포인트

이 파일은 Telethon 종속이므로 `infra/telegram` 아래에 두는 것이 맞습니다.
외부 SDK를 직접 DB 저장 함수가 알지 않도록 중간 완충 역할을 합니다.

## 7. `src/workers/collector_worker.py`  **[기존 보강]**

README상 이 파일은 실시간 텔레그램 메시지 이벤트 핸들러와 DB 저장 루프 역할입니다. STEP 1에서는 전체 완성이 아니라 **공통 저장 경로 호출 구조만** 먼저 심으면 됩니다. 

### 이번 단계에서 넣을 것

* `persist_message_from_event(...)`
* `handle_new_message(event)`
* `handle_edited_message(event)`
* `safe_process_event(...)`

### 핵심 흐름

1. 이벤트 수신
2. 채널 ACTIVE 여부 확인
3. translator로 payload 변환
4. repository `upsert_message()`
5. 성공 시 `update_channel_checkpoint()`
6. 실패 시 로그

### 구현 초안

```python
async def persist_message_from_event(db: AsyncSession, event_message, telegram_channel_id: int):
    payload = message_to_payload(event_message, telegram_channel_id)
    result = await upsert_message(db, payload)

    if result.action in {UpsertAction.CREATED, UpsertAction.UPDATED, UpsertAction.UNCHANGED}:
        await update_channel_checkpoint(
            db=db,
            telegram_channel_id=telegram_channel_id,
            candidate_message_id=result.checkpoint_candidate,
            collected_at=payload.collected_at,
        )
    return result
```

### 포인트

이 함수가 나중에 **백필에서도 그대로 재사용**되도록 만드는 것이 중요합니다.

# 추천 구현 순서

## 1순위

`src/infra/db/schemas.py`
`src/infra/db/repository.py`

이 둘이 먼저입니다.
타입 계약과 upsert 규약이 먼저 닫혀야 나머지가 흔들리지 않습니다.

`src/infra/telegram/translators.py`
`src/infra/telegram/client.py`

외부 Telethon 객체를 내부 payload로 바꾸는 경로를 닫습니다.

## 3순위

`src/web/routes/telegram.py`

`/sync`, `/toggle`를 붙입니다.

## 4순위

`src/workers/startup_tasks.py`
`src/workers/collector_worker.py`

실시간/백필 공통 저장 경로를 연결합니다.

## 5순위

`tests/*`

저장 규약과 상태 전이를 고정합니다.
