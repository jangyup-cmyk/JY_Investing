좋습니다.
그럼 바로 **다음 단계 구현 초안**으로 이어가겠습니다.

이번에는 아래 5가지를 붙여서 정리하겠습니다.

1. `inference_service.py`
2. `inference_worker.py`
3. `run_worker.py` smoke test
4. `scheduler_worker.py` 테스트 3개
5. 적용 순서와 체크포인트

이 단계까지 들어가면, 이제 정말로 **NORMALIZE → SINGLE_MESSAGE_SUMMARY → stuck recovery**의 핵심 루프가 코드 수준에서 거의 닫히게 됩니다.

# 1. `src/services/inference_service.py`

이 서비스는 `SINGLE_MESSAGE_SUMMARY` job을 실제로 수행합니다.

핵심 원칙은 딱 하나입니다.

**InferenceWorker는 raw message를 직접 읽지 않고, 반드시 preprocess 결과를 입력으로 사용한다.**

아래 초안은 그 원칙을 그대로 반영한 형태입니다.

```python
from __future__ import annotations

from dataclasses import dataclass
from typing import Any


@dataclass
class SummaryModelOutput:
    summary_text: str
    keywords: list[str] | None = None
    entities: list[dict[str, Any]] | None = None
    sentiment_label: str | None = None
    confidence_score: float | None = None
    raw_response: dict[str, Any] | None = None


class InferenceService:
    MODEL_NAME = "single-message-summary-model"
    PROMPT_VERSION = "summary-v1"
    TASK_TYPE = "SINGLE_MESSAGE_SUMMARY"

    def __init__(
        self,
        preprocess_result_repository,
        inference_result_repository,
        model_client,
    ):
        self.preprocess_result_repository = preprocess_result_repository
        self.inference_result_repository = inference_result_repository
        self.model_client = model_client

    def run_single_message_summary_job(self, session, *, job) -> int:
        payload = job.payload_json or {}
        preprocess_result_id = payload.get("preprocess_result_id")
        if not preprocess_result_id:
            raise ValueError("preprocess_result_id missing in job payload")

        preprocess_result = self.preprocess_result_repository.get_by_id(
            session,
            preprocess_result_id,
        )
        if preprocess_result is None:
            raise ValueError(
                f"preprocess_result not found: preprocess_result_id={preprocess_result_id}"
            )

        prompt = self._build_prompt(preprocess_result.normalized_text)
        model_output = self.model_client.summarize(prompt)

        parsed = self._parse_model_output(model_output)

        inference_result = self.inference_result_repository.create_result(
            session,
            message_id=job.message_id,
            message_version_id=job.message_version_id,
            preprocess_result_id=preprocess_result.id,
            inference_job_id=job.id,
            task_type=self.TASK_TYPE,
            model_name=self.MODEL_NAME,
            prompt_version=self.PROMPT_VERSION,
            summary_text=parsed.summary_text,
            keywords_json=parsed.keywords,
            entities_json=parsed.entities,
            sentiment_label=parsed.sentiment_label,
            confidence_score=parsed.confidence_score,
            raw_response_json=parsed.raw_response,
        )
        return inference_result.id

    def _build_prompt(self, normalized_text: str) -> str:
        return (
            "다음 텔레그램 메시지를 분석하라.\n"
            "1) 한 줄 요약\n"
            "2) 핵심 키워드 목록\n"
            "3) 주요 엔티티가 있으면 추출\n"
            "4) 메시지 톤(positive/neutral/negative)이 있으면 분류\n\n"
            f"[입력 메시지]\n{normalized_text}"
        )

    def _parse_model_output(self, model_output) -> SummaryModelOutput:
        """
        model_client가 dict를 반환한다고 가정한 파서.
        운영 중 필요하면 pydantic 검증으로 강화하는 것이 좋습니다.
        """
        if not isinstance(model_output, dict):
            raise ValueError("model output must be dict")

        summary_text = model_output.get("summary_text")
        if not summary_text or not isinstance(summary_text, str):
            raise ValueError("summary_text missing in model output")

        return SummaryModelOutput(
            summary_text=summary_text.strip(),
            keywords=model_output.get("keywords"),
            entities=model_output.get("entities"),
            sentiment_label=model_output.get("sentiment_label"),
            confidence_score=model_output.get("confidence_score"),
            raw_response=model_output,
        )
```

# 2. `src/workers/inference_worker.py`

이 worker는 `SINGLE_MESSAGE_SUMMARY` job 전용 소비자입니다.

구조는 `PreprocessWorker`와 거의 동일하되, 서비스만 `InferenceService`로 바뀝니다.

```python
from __future__ import annotations

import logging
import socket
import time

from src.domain.constants import JobType

logger = logging.getLogger(__name__)


class InferenceWorker:
    def __init__(
        self,
        session_factory,
        job_service,
        inference_service,
        poll_interval: float = 1.0,
    ):
        self.session_factory = session_factory
        self.job_service = job_service
        self.inference_service = inference_service
        self.poll_interval = poll_interval
        self.worker_id = f"inference:{socket.gethostname()}"

    def run_forever(self) -> None:
        while True:
            processed = self.run_once()
            if not processed:
                time.sleep(self.poll_interval)

    def run_once(self) -> bool:
        with self.session_factory() as session:
            job = self.job_service.claim_next_job(
                session,
                job_type=JobType.SINGLE_MESSAGE_SUMMARY.value,
                worker_id=self.worker_id,
            )

            if job is None:
                session.commit()
                return False

            try:
                inference_result_id = self.inference_service.run_single_message_summary_job(
                    session,
                    job=job,
                )
                self.job_service.mark_job_succeeded(session, job=job)
                session.commit()

                logger.info(
                    "inference job succeeded job_id=%s message_id=%s inference_result_id=%s",
                    job.id,
                    job.message_id,
                    inference_result_id,
                )
                return True

            except Exception as exc:
                self.job_service.mark_job_failed(
                    session,
                    job=job,
                    error_code="INFERENCE_ERROR",
                    error_message=str(exc),
                    retryable=True,
                )
                session.commit()

                logger.exception(
                    "inference job failed job_id=%s message_id=%s",
                    job.id,
                    job.message_id,
                )
                return True
```

### 이 worker의 핵심 포인트

* `NORMALIZE` job을 절대 건드리지 않음
* `job.payload_json["preprocess_result_id"]`를 신뢰 기준으로 삼음
* 실패해도 `commit()`해서 stuck 상태를 남기지 않음

# 5. `run_worker.py` smoke test 초안

사용자께서 앞서 원하신 흐름과 맞물려, `run_worker.py` smoke test는 매우 현실적인 가치가 있습니다.

여기서는 실제 worker 내부 동작까지 다 보는 게 아니라, **라우팅이 정상인지**만 확인하는 수준이면 충분합니다.

가정:

* `src.run_worker.main()` 존재
* 내부에서 `build_container()` 호출
* container가 `preprocess_worker`, `inference_worker`, `scheduler_worker` 팩토리를 제공

# 8. bootstrap 연결 시 최소 필요 사항

`build_container()`는 복잡하게 시작할 필요 없습니다.
아래 의존성만 맞으면 됩니다.

### container가 반환할 것

* `analysis_job_repository`
* `analysis_job_service`
* `preprocess_result_repository`
* `inference_result_repository`
* `model_client`
* `preprocess_service`
* `inference_service`
* `scheduler_service`
* `preprocess_worker`
* `inference_worker`
* `scheduler_worker`

예시 구조는 이런 느낌이면 충분합니다.

```python
class Container:

        self.analysis_job_repository = AnalysisJobRepository()
        self.preprocess_result_repository = PreprocessResultRepository()
        self.inference_result_repository = InferenceResultRepository()

        self.analysis_job_service = AnalysisJobService(self.analysis_job_repository)
        self.model_client = SummaryModelClientImpl()

        self.preprocess_service = PreprocessService(

        return SchedulerWorker(
            session_factory=self.session_factory,
            scheduler_service=self.scheduler_service,
        )
```

# 9. 여기까지 적용되면 실제 파이프라인은 이렇게 닫힙니다

1. `persist_message()`가 `NORMALIZE` job 생성
2. `PreprocessWorker`가 `NORMALIZE` claim
3. `PreprocessService`가 preprocess result 저장
4. `SINGLE_MESSAGE_SUMMARY` job 생성
5. `InferenceWorker`가 summary job claim
6. `InferenceService`가 inference result 저장
7. `SchedulerService`가 stuck job 복구

즉, 이제 구조상 빠져 있는 큰 조각은 거의 없습니다.

# 10. 지금 가장 자연스러운 다음 작업

이 시점에서 다음 우선순위는 아래 순서가 가장 좋습니다.

### 1순위

`persist_message()` 실제 코드 패치
