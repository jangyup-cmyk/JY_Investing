[프로젝트 시작 방법(한번에 한줄씩)]
(cmux 키고)

1. 프로젝트 폴더 만들기 
mkdir ~/Projects
cd Projects
mkdir {project-name}
cd ~/Projects/{project-name}

2. omx 키기
omx --high --madmax

이 아래 내용은 omx의 요청 입력창에 입력

3. deep-interview (내가 뭘 원하는지 구체화)
$deep-interview

나 {프로젝트 설명} 만들 건데 구체화하기 위해 인터뷰해줘.

나는 개발 지식 하나도 없는 초등학생 수준이니,
선택지를 줄 때마다 친절하고 자세한 설명을 덧붙여줘.

목표:
- 내가 만들고 싶은 제품의 핵심 목적을 명확히 하기
- 사용자, 주요 기능, 화면, 데이터, 운영 방식 정리하기
- MVP 범위와 나중에 할 일을 나누기
- 개발자가 바로 이해할 수 있는 요구사항으로 정리하기

주의:
- 질문은 한 번에 너무 많이 하지 말고, 단계별로 물어봐.
- 내가 애매하게 답하면 구체적인 선택지를 줘.

인터뷰가 끝나면 아래 2가지를 출력해줘:
1. 최종 요구사항 정리
2. 다음 단계인 $ralplan에 그대로 붙여넣을 프롬프트

위의 2가지 출력사항 interview-summary.md 파일로 저장하고 full path 줘

4. ralplan (개발 계획 문서 생산)
$ralplan 

[위에서 받은 $ralplan 프롬프트 복붙 ]

이번 개발 계획 spec문서 만들어 저장하고 다음 단계인 $ralph에 그대로 붙여넣을 프롬프트 써줘 


조건:
- 나는 개발 지식이 거의 없으니 설명은 쉽게 써줘.
- 하지만 문서는 개발자가 바로 구현할 수 있을 만큼 구체적으로 써줘.
- 불확실한 부분은 “결정 필요”로 표시해줘.
- 과한 기능은 MVP 이후로 분리해줘.
- 각 문서는 실제 파일로 생성해줘.

brew install mosh

그다음 tailscale 앱 켜서 로그인하고 ip주소 확인하기

sudo systemsetup -setremotelogin on

3. tailscale에서 내 컴퓨터에 부여해준 ip 확인
tailscale ip -4
이거 치거나 그냥 설치한 tailscale 앱 켜도 나옴

mosh user@100.xxx.xxx.xxx
- "user" 자리에 내 컴퓨터 user명 입력
- "100.xxx.xxx.xxx" 자리에 tailscale ip -4 에서 나온 ip주소 입력
