### 이 파일이 잠그는 규약

* `PENDING_BACKFILL` 상태 채널만 백필 대상으로 잡힘
* 백필 성공 시 `ACTIVE` 전이 + `last_backfill_at` 기록
* 백필 실패 시 `ERROR` 전이 + 에러 메시지 기록
* `perform_backfill_for_channel()`은 최근 3일 이내 메시지만 처리
* cutoff보다 오래된 메시지를 만나면 중단
* naive datetime도 UTC 기준으로 안전하게 처리

### 가장 먼저 깨질 가능성이 높은 부분

아마 둘 중 하나입니다.

* `TelegramChannelCollectionState` 필드명이 `telegram_channel_id`가 아니라 `channel_id`인 경우
* `mark_backfill_completed()`가 `last_error_at`까지 `None`으로 지우지 않는 경우

첫 번째는 필드명만 바꾸면 되고,
두 번째는 주석 처리한 assertion처럼 정책에 맞춰 한 줄만 조정하면 됩니다.

다음으로 가장 자연스러운 작업은 **이 테스트 기준으로 `startup_tasks.py` 구현이 어디서 먼저 깨질지 점검**하는 것입니다.

네. 지금 `tests/test_startup_tasks.py` 기준으로 보면, **가장 먼저 깨질 가능성이 높은 순서**는 아래입니다.

## 1) `perform_backfill_for_channel`가 아직 파일에 없습니다

가장 먼저는 이겁니다.
현재 `startup_tasks.py`에는 `find_channels_pending_backfill`, `run_startup_recovery`, `mark_backfill_completed`, `mark_backfill_failed`는 보이지만, **`perform_backfill_for_channel` 함수 자체가 아직 정의되어 있지 않습니다.** 그래서 제가 드린 테스트 전체본을 그대로 넣으면, 테스트 실행 전에 import 단계에서 바로 실패할 가능성이 큽니다. `backfill_missed_messages()` 안에도 실제 백필 호출은 TODO로 남아 있습니다.

## 2) `mark_backfill_completed()`와 `mark_backfill_failed()`는 아직 `pass` 상태입니다

두 함수 모두 현재 구현 예정 상태라서, 테스트가 기대하는

* `ACTIVE` 전이
* `ERROR` 전이
* `last_backfill_at`
* `last_error_message`
  기록이 전혀 일어나지 않습니다.
  즉 import가 해결되면 다음으로 이 두 테스트가 바로 깨질 가능성이 높습니다.

## 3) `run_startup_recovery()`도 아직 `pass`입니다

이건 당장 핵심 테스트 대상은 아니더라도, startup recovery 관련 테스트를 넣기 시작하면 바로 실패합니다. 현재는 진입점만 있고 실제 동작은 없습니다.

## 4) `find_channels_pending_backfill()`는 상대적으로 통과 가능성이 높습니다

이 함수는 현재 구현이 들어가 있고, 조인도
`TelegramChannel.id == TelegramChannelCollectionState.telegram_channel_id`
기준으로 되어 있습니다. 지금 테스트도 같은 전제를 쓰면 통과 가능성이 있습니다. 다만 **실제 모델 FK 컬럼명이 정말 `telegram_channel_id`인지**는 꼭 확인해야 합니다. repository도 현재 `update_channel_checkpoint()`에서 `TelegramChannelCollectionState.telegram_channel_id == channel_id`를 쓰고 있어서, 지금 프로젝트는 일단 이 기준으로 맞춰져 있는 쪽에 가깝습니다.

## 5) 다음 단계 구현 시 `collector_worker.py`와 시그니처가 같이 깨질 수 있습니다

이건 테스트보다 실제 런타임 이슈입니다.
현재 `collector_worker.py`는 아직 `await run_startup_tasks()`를 **인자 없이** 호출하고 있고, 이벤트도 `self._collector.add_event_handler(...)`를 직접 부르고 있습니다. 반면 제가 제안한 STEP 2 초안은 `run_startup_tasks(collector, persist_func)`와 client wrapper 메서드 사용을 전제로 합니다. 그래서 `startup_tasks.py`를 먼저 STEP 2 방식으로 바꾸면, 그다음엔 `collector_worker.py`도 **같이** 수정해야 합니다.

## 6) 백필 구분(`is_backfill=True`)도 아직 worker에 반영되지 않았습니다

현재 `persist_message_payload()`는 checkpoint 갱신 시 **항상 `is_backfill=False`**로 호출합니다. 그래서 나중에 `perform_backfill_for_channel()`를 붙이더라도, worker를 같이 손보지 않으면 `last_backfill_at` 같은 메타데이터가 제대로 안 남을 수 있습니다. `persist_message_from_history()`도 현재는 이벤트 경로와 동일한 래퍼만 호출합니다.

# 결론

지금 테스트 기준으로 **실패 우선순위**를 한 줄로 정리하면 이렇습니다.

1. **`perform_backfill_for_channel` 미정의**로 import 단계 실패
2. `mark_backfill_completed()` / `mark_backfill_failed()`가 `pass`라서 상태 전이 테스트 실패
3. 그다음 실제 백필 흐름 연결 시 `collector_worker.py` 시그니처 수정 필요

즉, 지금 가장 먼저 해야 할 건 테스트를 돌려보기 전에 **`startup_tasks.py`에 아래 3개를 실제로 넣는 것**입니다.

* `perform_backfill_for_channel()`
* `mark_backfill_completed()`
* `mark_backfill_failed()`

그 다음에 `collector_worker.py`를

* `run_startup_tasks(collector=..., persist_func=...)`
* `persist_message_payload(..., is_backfill=True/False)`
  구조로 같이 맞추는 순서가 가장 자연스럽습니다.

원하시면 다음 답변에서 바로 **`startup_tasks.py`에 들어갈 최소 구현본**을, 현재 파일 기준으로 충돌 없이 붙여넣을 수 있는 형태로 다시 정리해드리겠습니다.

좋습니다.
현재 구조와 **충돌을 최소화하는 최소 구현본**으로 드리겠습니다.

핵심은 두 가지입니다.

* `collector_worker.py`가 아직 `run_startup_tasks()`를 **인자 없이** 호출하고 있으므로, 지금 단계에서는 `collector=None, persist_func=None`을 허용해 **하위 호환**으로 두는 게 안전합니다. 
* 현재 `startup_tasks.py`에서 실제로 비어 있는 핵심은 `perform_backfill_for_channel`, `mark_backfill_completed`, `mark_backfill_failed`, `run_startup_recovery`입니다. 

아래 코드는 **`startup_tasks.py` 전체 교체용 최소 구현본**입니다.

```python
from datetime import datetime, timezone, timedelta

from sqlalchemy import select

from src.common.logger import logger
from src.infra.db.session import get_session
from src.infra.db.models import TelegramChannel, TelegramChannelCollectionState
from src.infra.telegram.schemas import ChannelCollectionStatus


BACKFILL_DAYS = 3
BACKFILL_LIMIT = 1000


def _status_value(status) -> str:
    return status.value if hasattr(status, "value") else str(status)


async def run_startup_tasks(collector=None, persist_func=None) -> None:
    """워커 시작 시 필요한 자동화 작업을 순차적으로 수행합니다."""
    logger.info("Startup 자동화 작업 시작...")

    active_count = await restore_last_selected_channels()
    logger.info(f"수집 활성화된 채널 수: {active_count}")

    await run_startup_recovery()
    await backfill_missed_messages(
        collector=collector,
        persist_func=persist_func,
    )

    logger.info("Startup 자동화 작업 완료")


async def sync_channels_on_startup() -> None:
    """설정에 따라 텔레그램 채널 목록을 자동으로 업데이트합니다."""
    logger.info("자동 채널 업데이트 스킵 (UI에서 수동 실행 필요)")


async def restore_last_selected_channels() -> int:
    """마지막으로 선택된 채널들의 개수를 반환합니다."""
    async with get_session() as db:
        stmt = select(TelegramChannel).where(TelegramChannel.is_collecting.is_(True))
        result = await db.execute(stmt)
        channels = result.scalars().all()

        for c in channels:
            logger.info(f"수집 복원 대상 채널: [{c.title}] ({c.tg_channel_id})")

        return len(channels)


async def backfill_missed_messages(collector=None, persist_func=None) -> None:
    """
    [백필(Backfill) 핵심 원칙]
    - 대상: 수집 활성화 상태이면서 PENDING_BACKFILL 상태인 채널
    - 범위: 최근 3일 또는 최근 1,000건
    - 동작: 백필 완료 후 ACTIVE 전이
    """
    logger.info("PENDING_BACKFILL 상태의 채널 스캔 시작...")

    async with get_session() as db:
        pending_channels = await find_channels_pending_backfill(db)

        if not pending_channels:
            logger.info("현재 백필이 필요한 채널이 없습니다.")
            return

        logger.info(f"총 {len(pending_channels)}개의 채널이 백필 대기 중입니다.")

        # 아직 worker/collector 연결 전이면 스캔만 하고 종료
        if collector is None or persist_func is None:
            logger.info("collector/persist_func 미주입 상태이므로 실제 백필은 건너뜁니다.")
            return

        for channel, state in pending_channels:
            logger.info(f"백필 대상 채널: [{channel.title}] ({channel.tg_channel_id})")

            try:
                processed_count = await perform_backfill_for_channel(
                    db=db,
                    collector=collector,
                    channel=channel,
                    persist_func=persist_func,
                    days=BACKFILL_DAYS,
                    limit=BACKFILL_LIMIT,
                )

                await mark_backfill_completed(db, channel.id)
                await db.commit()

                logger.info(
                    f"백필 완료 처리: [{channel.title}] ({channel.tg_channel_id}) "
                    f"messages={processed_count}"
                )

            except Exception as e:
                await db.rollback()

                # rollback 이후 동일 세션으로 새 트랜잭션 시작
                await mark_backfill_failed(db, channel.id, str(e))
                await db.commit()

                logger.error(
                    f"백필 실패 처리: [{channel.title}] ({channel.tg_channel_id}) error={e}"
                )


async def perform_backfill_for_channel(
    db,
    collector,
    channel,
    persist_func,
    *,
    days: int = 3,
    limit: int = 1000,
) -> int:
    """
    단일 채널에 대해 최근 N일 또는 최대 N건까지 히스토리를 읽어 공통 persistence 경로로 적재합니다.
    """
    cutoff = datetime.now(timezone.utc) - timedelta(days=days)
    processed_count = 0

    async for history_message in collector.iter_channel_messages(
        channel.tg_channel_id,
        limit=limit,
        days=days,
    ):
        msg_date = getattr(history_message, "date", None)

        if msg_date is not None:
            if msg_date.tzinfo is None:
                msg_date = msg_date.replace(tzinfo=timezone.utc)
            else:
                msg_date = msg_date.astimezone(timezone.utc)

            if msg_date < cutoff:
                break

        await persist_func(db, history_message, channel.id)
        processed_count += 1

        if processed_count % 100 == 0:
            logger.info(f"백필 진행중: [{channel.title}] count={processed_count}")

    return processed_count


async def find_channels_pending_backfill(db) -> list:
    """PENDING_BACKFILL 상태인 채널(channel, state 튜플)을 조회합니다."""
    stmt = (
        select(TelegramChannel, TelegramChannelCollectionState)
        .join(
            TelegramChannelCollectionState,
            TelegramChannel.id == TelegramChannelCollectionState.telegram_channel_id,
        )
        .where(
            TelegramChannel.is_collecting.is_(True),
            TelegramChannelCollectionState.status
            == _status_value(ChannelCollectionStatus.PENDING_BACKFILL),
        )
    )
    result = await db.execute(stmt)
    return result.all()


async def run_startup_recovery() -> None:
    """워커 시작 시 복구 절차를 수행합니다."""
    async with get_session() as db:
        pending_channels = await find_channels_pending_backfill(db)

        if pending_channels:
            logger.info(f"스타트업 복구: 백필 대기 채널 {len(pending_channels)}개 확인")
        else:
            logger.info("스타트업 복구: 백필 대기 채널 없음")


async def mark_backfill_completed(db, channel_id: int) -> None:
    """백필 완료 시 ACTIVE로 전환하고 에러 상태를 정리합니다."""
    stmt = select(TelegramChannelCollectionState).where(
        TelegramChannelCollectionState.telegram_channel_id == channel_id
    )
    result = await db.execute(stmt)
    state = result.scalar_one_or_none()

    now = datetime.now(timezone.utc)

    if state is None:
        state = TelegramChannelCollectionState(
            telegram_channel_id=channel_id,
            status=_status_value(ChannelCollectionStatus.ACTIVE),
            last_backfill_at=now,
            last_error_message=None,
            last_error_at=None,
        )
        db.add(state)
    else:
        state.status = _status_value(ChannelCollectionStatus.ACTIVE)
        state.last_backfill_at = now
        state.last_error_message = None
        state.last_error_at = None


async def mark_backfill_failed(db, channel_id: int, error_message: str) -> None:
    """백필 실패 시 ERROR로 전환하고 에러 메시지를 기록합니다."""
    stmt = select(TelegramChannelCollectionState).where(
        TelegramChannelCollectionState.telegram_channel_id == channel_id
    )
    result = await db.execute(stmt)
    state = result.scalar_one_or_none()

    now = datetime.now(timezone.utc)

    if state is None:
        state = TelegramChannelCollectionState(
            telegram_channel_id=channel_id,
            status=_status_value(ChannelCollectionStatus.ERROR),
            last_error_message=error_message,
            last_error_at=now,
        )
        db.add(state)
    else:
        state.status = _status_value(ChannelCollectionStatus.ERROR)
        state.last_error_message = error_message
        state.last_error_at = now
```

## 이 버전의 장점

* **지금 즉시 적용 가능**합니다.
  `collector_worker.py`를 아직 안 바꿔도 `run_startup_tasks()` 무인자 호출이 깨지지 않습니다. 
* `tests/test_startup_tasks.py`에서 기대한

  * `perform_backfill_for_channel()`
  * `mark_backfill_completed()`
  * `mark_backfill_failed()`
    를 바로 검증할 수 있습니다. 
* 이후 STEP 2에서 `collector`와 `persist_func`를 넘기기만 하면 **실제 백필 실행**으로 자연스럽게 확장됩니다.
