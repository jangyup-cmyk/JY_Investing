좋습니다.
이제부터는 **실제 파일 단위 적용안**으로 내려가겠습니다.

즉, 아래 파일들에 대해

* 어떤 클래스/함수를 추가할지
* 기존 흐름에서 무엇을 제거할지
* 어떤 순서로 연결할지
* 테스트는 어디에 붙일지

를 **코드 적용 관점**으로 구체화하겠습니다.

# 1. 최종적으로 바뀌는 파일 구조

권장 구조는 아래처럼 가져가면 가장 안정적입니다.

```text
src/
  db/
    models/
      message.py
      message_version.py
      analysis_job.py
      preprocess_result.py
      inference_result.py

  repositories/
    message_repository.py
    message_version_repository.py
    analysis_job_repository.py
    preprocess_result_repository.py
    inference_result_repository.py

  services/
    message_persistence_service.py
    analysis_job_service.py
    preprocess_service.py
    inference_service.py
    scheduler_service.py

  workers/
    collector_worker.py
    preprocess_worker.py
    inference_worker.py
    scheduler_worker.py

  run_worker.py
```

이미 구조가 다르면 파일명은 유지해도 됩니다.
중요한 건 **책임 분리**입니다.

# 2. 먼저 공통 상수부터 고정하는 것이 좋습니다

상태 문자열과 job type 문자열이 여기저기 흩어지면 반드시 깨집니다.
그래서 제일 먼저 공통 enum 또는 상수 파일을 두는 게 좋습니다.

예:

```python
# src/domain/constants.py
from enum import Enum


class JobType(str, Enum):
    NORMALIZE = "NORMALIZE"
    SINGLE_MESSAGE_SUMMARY = "SINGLE_MESSAGE_SUMMARY"


class JobStatus(str, Enum):
    PENDING = "PENDING"
    IN_PROGRESS = "IN_PROGRESS"
    SUCCEEDED = "SUCCEEDED"
    FAILED = "FAILED"
    RETRY_WAIT = "RETRY_WAIT"
    CANCELLED = "CANCELLED"


class MessageStatus(str, Enum):
    ACTIVE = "ACTIVE"
    DELETED = "DELETED"
    SKIPPED = "SKIPPED"
```

이걸 먼저 해두면 이후 모든 파일이 깔끔해집니다.

## 5-1. `message_persistence_service.py`

이 서비스가 사실상 **수집부의 핵심 진입점**입니다.

### 책임

* 메시지 신규/수정 판별
* 해시 비교
* 버전 저장
* `NORMALIZE` job 생성
* 트랜잭션 보장

### 핵심 메서드

* `persist_message(channel, telegram_message)`

예시 골격:

```python
# src/services/message_persistence_service.py
import hashlib
from dataclasses import dataclass
from datetime import datetime, timezone

from src.domain.constants import JobType
from src.repositories.message_repository import MessageRepository
from src.repositories.message_version_repository import MessageVersionRepository
from src.repositories.analysis_job_repository import AnalysisJobRepository


@dataclass
class PersistMessageResult:
    message_id: int
    message_version_id: int | None
    created: bool
    updated: bool
    normalize_job_created: bool
    skipped_as_unchanged: bool


class MessagePersistenceService:
    def __init__(
        self,
        message_repo: MessageRepository,
        version_repo: MessageVersionRepository,
        job_repo: AnalysisJobRepository,
    ) -> None:
        self.message_repo = message_repo
        self.version_repo = version_repo
        self.job_repo = job_repo

    def persist_message(self, session, *, channel_id: int, telegram_message) -> PersistMessageResult:
        raw_text = telegram_message.text or ""
        text_hash = self._hash_text(raw_text)
        now = datetime.now(timezone.utc)

        existing = self.message_repo.get_by_channel_and_tg_message_id(
            session,
            channel_id=channel_id,
            telegram_message_id=telegram_message.id,
        )

        if existing is None:
            message = self.message_repo.create_message(
                session,
                channel_id=channel_id,
                telegram_message_id=telegram_message.id,
                posted_at=telegram_message.date,
                edited_at=getattr(telegram_message, "edit_date", None),
                raw_text=raw_text,
                has_media=bool(getattr(telegram_message, "media", None)),
            )
            version = self.version_repo.create_version(
                session,
                message_id=message.id,
                version_no=1,
                raw_text=raw_text,
                edited_at=getattr(telegram_message, "edit_date", None),
                text_hash=text_hash,
                change_type="INITIAL",
            )
            self.message_repo.update_latest_version(
                session,
                message=message,
                version_id=version.id,
                version_no=1,
                raw_text=raw_text,
                edited_at=getattr(telegram_message, "edit_date", None),
            )
            self.job_repo.enqueue_job(
                session,
                job_type=JobType.NORMALIZE.value,
                message_id=message.id,
                message_version_id=version.id,
            )
            return PersistMessageResult(
                message_id=message.id,
                message_version_id=version.id,
                created=True,
                updated=False,
                normalize_job_created=True,
                skipped_as_unchanged=False,
            )

        latest_version = self.version_repo.get_latest_version(session, existing.id)
        if latest_version and latest_version.text_hash == text_hash:
            return PersistMessageResult(
                message_id=existing.id,
                message_version_id=None,
                created=False,
                updated=False,
                normalize_job_created=False,
                skipped_as_unchanged=True,
            )

        next_version_no = (existing.latest_version_no or 0) + 1
        version = self.version_repo.create_version(
            session,
            message_id=existing.id,
            version_no=next_version_no,
            raw_text=raw_text,
            edited_at=getattr(telegram_message, "edit_date", None),
            text_hash=text_hash,
            change_type="EDIT",
        )
        self.message_repo.update_latest_version(
            session,
            message=existing,
            version_id=version.id,
            version_no=next_version_no,
            raw_text=raw_text,
            edited_at=getattr(telegram_message, "edit_date", None),
        )
        self.job_repo.enqueue_job(
            session,
            job_type=JobType.NORMALIZE.value,
            message_id=existing.id,
            message_version_id=version.id,
        )
        return PersistMessageResult(
            message_id=existing.id,
            message_version_id=version.id,
            created=False,
            updated=True,
            normalize_job_created=True,
            skipped_as_unchanged=False,
        )

    @staticmethod
    def _hash_text(text: str) -> str:
        return hashlib.sha256(text.encode("utf-8")).hexdigest()
```

## 5-2. `analysis_job_service.py`

이 서비스는 상태 전이 일관성을 담당합니다.

예시 골격:

```python
from datetime import datetime, timedelta, timezone

from src.domain.constants import JobStatus


class AnalysisJobService:
    def __init__(self, job_repo):
        self.job_repo = job_repo

    def enqueue_job(self, session, *, job_type: str, message_id: int, message_version_id: int, payload_json: dict | None = None):
        return self.job_repo.enqueue_job(
            session,
            job_type=job_type,
            message_id=message_id,
            message_version_id=message_version_id,
            payload_json=payload_json,
        )

    def claim_next_job(self, session, *, job_type: str, worker_id: str):
        now = datetime.now(timezone.utc)
        return self.job_repo.claim_next_job(session, job_type=job_type, worker_id=worker_id, now=now)

    def mark_job_succeeded(self, session, *, job):
        now = datetime.now(timezone.utc)
        self.job_repo.mark_succeeded(session, job=job, now=now)

    def mark_job_failed(self, session, *, job, error_code: str, error_message: str, retryable: bool):
        now = datetime.now(timezone.utc)
        if retryable and job.retry_count + 1 < job.max_retries:
            next_available_at = now + self._compute_backoff(job.retry_count + 1)
            self.job_repo.mark_retry_wait(
                session,
                job=job,
                now=now,
                next_available_at=next_available_at,
                error_code=error_code,
                error_message=error_message,
            )
        else:
            self.job_repo.mark_failed(
                session,
                job=job,
                now=now,
                error_code=error_code,
                error_message=error_message,
            )

    def touch_heartbeat(self, session, *, job):
        now = datetime.now(timezone.utc)
        self.job_repo.touch_heartbeat(session, job=job, now=now)

    @staticmethod
    def _compute_backoff(retry_count: int) -> timedelta:
        if retry_count == 1:
            return timedelta(minutes=1)
        if retry_count == 2:
            return timedelta(minutes=5)
        return timedelta(minutes=15)
```

```python
        self,
        version_repo,
        preprocess_result_repo,
        job_service,
    ):
        self.version_repo = version_repo
        self.preprocess_result_repo = preprocess_result_repo
        self.job_service = job_service

    def run_normalize_job(self, session, *, job):
        message_version = self.version_repo.get_by_id(session, job.message_version_id)
        if message_version is None:
            raise ValueError(f"message_version not found: {job.message_version_id}")

        output = self._normalize(message_version.raw_text or "")

        preprocess_result = self.preprocess_result_repo.create_result(
            session,
            message_id=job.message_id,
            message_version_id=message_version.id,
            normalize_job_id=job.id,
            normalized_text=output.normalized_text,
            cleaned_text=output.cleaned_text,
            language_code=output.language_code,
            token_count=output.token_count,
            line_count=output.line_count,
            char_count=output.char_count,
            has_url=output.has_url,
            has_ticker_hint=output.has_ticker_hint,
            preprocess_version=self.PREPROCESS_VERSION,
        )

        self.job_service.enqueue_job(
            session,
            job_type=JobType.SINGLE_MESSAGE_SUMMARY.value,
            message_id=job.message_id,
            message_version_id=message_version.id,
            payload_json={"preprocess_result_id": preprocess_result.id},
        )

        return preprocess_result.id

    def _normalize(self, raw_text: str) -> NormalizeOutput:
        cleaned = raw_text.replace("\u200b", " ").strip()
        normalized = " ".join(cleaned.split())
        return NormalizeOutput(
            normalized_text=normalized,
            cleaned_text=cleaned,
            language_code=None,
            token_count=len(normalized.split()),
            line_count=len(cleaned.splitlines()) if cleaned else 0,
            char_count=len(normalized),
            has_url=("http://" in normalized or "https://" in normalized),
            has_ticker_hint=any(token.startswith("$") for token in normalized.split()),
        )
```

### 핵심 포인트

전처리 워커는 여기서 **AI 추론을 하지 않습니다.**
오직 전처리 + 다음 job 생성만 합니다.

## 5-4. `inference_service.py`

이 서비스는 summary job 처리 전용입니다.

### 책임

* preprocess_result 조회
* prompt 구성
* 모델 호출
* 결과 저장

예시 골격:

```python
class InferenceService:
    MODEL_NAME = "your-model-name"
    PROMPT_VERSION = "summary-v1"
    TASK_TYPE = "SINGLE_MESSAGE_SUMMARY"

    def __init__(self, preprocess_result_repo, inference_result_repo, model_client):
        self.preprocess_result_repo = preprocess_result_repo
        self.inference_result_repo = inference_result_repo
        self.model_client = model_client

    def run_single_message_summary_job(self, session, *, job):
        preprocess_result_id = (job.payload_json or {}).get("preprocess_result_id")
        if not preprocess_result_id:
            raise ValueError("preprocess_result_id missing in job payload")

        preprocess_result = self.preprocess_result_repo.get_by_id(session, preprocess_result_id)
        if preprocess_result is None:
            raise ValueError(f"preprocess_result not found: {preprocess_result_id}")

        prompt = self._build_prompt(preprocess_result.normalized_text)
        model_output = self.model_client.summarize(prompt)

        inference_result = self.inference_result_repo.create_result(
            session,
            message_id=job.message_id,
            message_version_id=job.message_version_id,
            preprocess_result_id=preprocess_result.id,
            inference_job_id=job.id,
            task_type=self.TASK_TYPE,
            model_name=self.MODEL_NAME,
            prompt_version=self.PROMPT_VERSION,
            summary_text=model_output["summary_text"],
            keywords_json=model_output.get("keywords"),
            entities_json=model_output.get("entities"),
            sentiment_label=model_output.get("sentiment_label"),
            confidence_score=model_output.get("confidence_score"),
            raw_response_json=model_output,
        )
        return inference_result.id

    def _build_prompt(self, normalized_text: str) -> str:
        return f"다음 메시지를 한 줄로 요약하고 핵심 키워드를 추출하라:\n\n{normalized_text}"
```

### 핵심 포인트

이 서비스는 **반드시 preprocess 결과를 입력으로 사용**해야 합니다.
`messages.raw_text`를 다시 읽기 시작하면 버전 고정이 무너집니다.

## 5-5. `scheduler_service.py`

이 서비스는 stuck job 복구 전용입니다.

예시:

```python
from dataclasses import dataclass
from datetime import datetime, timedelta, timezone


@dataclass
class RecoveryStats:
    recovered_count: int
    failed_count: int


class SchedulerService:
    def __init__(self, job_repo):
        self.job_repo = job_repo

    def recover_stuck_jobs(self, session, *, timeout_minutes: int = 10) -> RecoveryStats:
        now = datetime.now(timezone.utc)
        timeout_before = now - timedelta(minutes=timeout_minutes)

        jobs = self.job_repo.find_stuck_jobs(session, timeout_before=timeout_before)

        recovered_count = 0
        failed_count = 0

        for job in jobs:
            if job.retry_count + 1 < job.max_retries:
                self.job_repo.mark_retry_wait(
                    session,
                    job=job,
                    now=now,
                    next_available_at=now + timedelta(minutes=1),
                    error_code="STUCK_TIMEOUT",
                    error_message="Job exceeded heartbeat timeout",
                )
                recovered_count += 1
            else:
                self.job_repo.mark_failed(
                    session,
                    job=job,
                    now=now,
                    error_code="STUCK_MAX_RETRIES_EXCEEDED",
                    error_message="Job exceeded heartbeat timeout and max retries",
                )
                failed_count += 1

        return RecoveryStats(recovered_count=recovered_count, failed_count=failed_count)
```

## 6-1. `collector_worker.py`

### 기존 구조에서 제거해야 할 것

수집 루프 안에서 아래가 있으면 제거 대상입니다.

* 직접 normalize 호출
* 직접 summary 호출
* 직접 inference 결과 저장

### 바뀐 구조

collector는 아래만 해야 합니다.

1. 텔레그램 메시지 읽기
2. `persist_message()` 호출
3. 필요하면 로깅

예시:

```python
# src/workers/collector_worker.py
class CollectorWorker:
    def __init__(self, session_factory, telegram_client, message_persistence_service):
        self.session_factory = session_factory
        self.telegram_client = telegram_client
        self.message_persistence_service = message_persistence_service

    def collect_channel(self, channel):
        for tg_message in self.telegram_client.iter_channel_messages(channel.tg_channel_id):
            with self.session_factory() as session:
                result = self.message_persistence_service.persist_message(
                    session,
                    channel_id=channel.id,
                    telegram_message=tg_message,
                )
                session.commit()
```

### 핵심 변화

예전에는 collector가 “수집 + 분석”이었다면, 이제는 “수집 + enqueue”입니다.

## 6-2. `preprocess_worker.py`

이제 이 워커는 `NORMALIZE` job 전용 소비자입니다.

예시:

```python
# src/workers/preprocess_worker.py
import logging
import socket
import time

from src.domain.constants import JobType

logger = logging.getLogger(__name__)


class PreprocessWorker:
    def __init__(self, session_factory, job_service, preprocess_service, poll_interval: float = 1.0):
        self.session_factory = session_factory
        self.job_service = job_service
        self.preprocess_service = preprocess_service
        self.poll_interval = poll_interval
        self.worker_id = f"preprocess:{socket.gethostname()}"

    def run_forever(self):
        while True:
            processed = self.run_once()
            if not processed:
                time.sleep(self.poll_interval)

    def run_once(self) -> bool:
        with self.session_factory() as session:
            job = self.job_service.claim_next_job(
                session,
                job_type=JobType.NORMALIZE.value,
                worker_id=self.worker_id,
            )
            if job is None:
                session.commit()
                return False

            try:
                self.preprocess_service.run_normalize_job(session, job=job)
                self.job_service.mark_job_succeeded(session, job=job)
                session.commit()
                return True
            except Exception as exc:
                self.job_service.mark_job_failed(
                    session,
                    job=job,
                    error_code="NORMALIZE_ERROR",
                    error_message=str(exc),
                    retryable=True,
                )
                session.commit()
                logger.exception("Failed to process normalize job id=%s", job.id)
                return True
```

### 핵심 포인트

* 메시지 직접 조회 loop 제거
* 반드시 `claim_next_job()` 사용
* 성공 시 `mark_job_succeeded()`
* 실패 시 `mark_job_failed()`

```python
class SchedulerWorker:
```

```python
# src/run_worker.py
import argparse

from src.bootstrap import build_container


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--worker", required=True, choices=["preprocess", "inference", "scheduler"])
    parser.add_argument("--once", action="store_true")
    args = parser.parse_args()

    container = build_container()

    if args.worker == "preprocess":

        worker = container.inference_worker()
    elif args.worker == "scheduler":
        worker = container.scheduler_worker()

        raise ValueError(f"Unsupported worker type: {args.worker}")

        worker.run_once()
    else:
        worker.run_forever()

    main()
```

# 10. 가장 많이 깨지는 지점 5개

실제 적용하면서 특히 주의할 점입니다.

---

## 1. collector가 아직도 직접 AI를 호출함

이건 반드시 제거해야 합니다.
collector는 enqueue만 해야 합니다.

---

## 2. `claim_next_job()`가 원자적이지 않음

멀티워커 환경이면 같은 job을 두 워커가 동시에 잡을 수 있습니다.
가능하면 `FOR UPDATE SKIP LOCKED`를 써야 합니다.

---

## 3. worker가 `messages.raw_text`를 다시 읽음

이러면 수정 메시지 버전 고정이 무너집니다.
반드시 `message_version_id` 기준으로 가야 합니다.

---

## 4. 실패 시 commit 누락

예외 처리 후 상태 바꿔놓고 commit 안 하면 job이 영원히 `IN_PROGRESS`로 남을 수 있습니다.

---

## 5. Scheduler timeout이 너무 짧음

정상 처리 중인 job까지 stuck으로 오인하면 중복 실행이 생깁니다.

---

# 11. 현재 코드베이스에 바로 대응되는 핵심 적용 문장

한 줄씩 정리하면 이렇습니다.

* `collector_worker.py`는 **저장 + job 생성까지만**
* `persist_message()`는 **message/version/job를 한 트랜잭션으로**
* `PreprocessWorker`는 **NORMALIZE job 소비**
* `InferenceWorker`는 **SINGLE_MESSAGE_SUMMARY job 소비**
* `SchedulerWorker`는 **stuck job 복구**
* `run_worker.py`는 **공통 실행 진입점**

---

# 12. 최종적으로 운영에서 기대되는 모습

이 구조가 들어가면 실제 실행 흐름은 이렇게 됩니다.

1. collector가 텔레그램 메시지 수집
2. `persist_message()`가 최신 메시지와 버전 저장
3. `NORMALIZE` job 생성
4. `PreprocessWorker`가 job claim 후 전처리
5. preprocess result 저장
6. `SINGLE_MESSAGE_SUMMARY` job 생성
7. `InferenceWorker`가 job claim 후 요약/추론
8. inference result 저장
9. worker 장애로 job이 멈추면 `SchedulerWorker`가 복구

즉, 지금부터는 정말로 **적용/테스트 단계**입니다.

---

다음으로 가장 자연스럽게 이어질 작업은
**“이 파일 구조를 기준으로 실제 코드 패치 초안”**, 즉
`analysis_job_service.py`, `preprocess_worker.py`, `scheduler_worker.py`, `run_worker.py` 수준의 **구체 코드 뼈대**를 더 이어서 적는 것입니다.
