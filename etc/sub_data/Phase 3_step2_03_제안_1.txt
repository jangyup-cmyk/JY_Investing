좋습니다. 아래는 **`tests/workers/test_preprocess_worker.py` 전체 초안**입니다.

이 파일의 목적은 세 가지입니다.

* `PreprocessWorker.run_once()`가 `normalize` job을 정상 소비하는지
* 그 결과로 `normalized_messages`와 다음 `extract_features` job이 생성되는지
* 이어서 `extract_features` job이 `message_features`와 `single_message_summary` job까지 연결하는지

즉, 지금 설계한 **Phase 3 전처리 파이프라인의 핵심 연결부**를 검증하는 테스트입니다.

async def count_rows(session: AsyncSession, model) -> int:
    return int(await session.scalar(select(func.count()).select_from(model)) or 0)


async def get_jobs_by_type(session: AsyncSession, job_type: JobType) -> list[AnalysisJob]:
    result = await session.scalars(
        select(AnalysisJob)
        .where(AnalysisJob.job_type == job_type)
        .order_by(AnalysisJob.id.asc())
    )
    return list(result)


async def get_message_jobs(session: AsyncSession, message_id: int, job_type: JobType) -> list[AnalysisJob]:
    result = await session.scalars(
        select(AnalysisJob)
        .where(
            AnalysisJob.target_type == JobTargetType.MESSAGE,
            AnalysisJob.target_id == message_id,
            AnalysisJob.job_type == job_type,
        )
        .order_by(AnalysisJob.id.asc())
    )
    return list(result)


async def get_latest_normalized(session: AsyncSession, message_id: int) -> NormalizedMessage | None:
    return await session.scalar(
        select(NormalizedMessage)
        .where(NormalizedMessage.message_id == message_id)
        .order_by(NormalizedMessage.id.desc())
    )


async def get_latest_feature(session: AsyncSession, message_id: int) -> MessageFeature | None:
    return await session.scalar(
        select(MessageFeature)
        .where(MessageFeature.message_id == message_id)
        .order_by(MessageFeature.id.desc())
    )

async def seed_normalize_job(session: AsyncSession, message: Message, version: MessageVersion) -> AnalysisJob:
    job = await AnalysisJobService.enqueue_job(
        session,
        job_type=JobType.NORMALIZE,
        target_type=JobTargetType.MESSAGE,
        target_id=message.id,
        idempotency_key=f"normalize:message:{message.id}:version:{version.id}:cv1",
        payload_json={
            "message_id": message.id,
            "version_id": version.id,
            "channel_id": message.channel_id,
            "tg_message_id": message.tg_message_id,
            "cleaning_version": "v1",
        },
        priority=100,
        max_attempts=3,
    )
    await session.commit()
    return job

---

### 이 파일에서 실제 프로젝트에 맞춰 조정할 부분

가장 가능성이 높은 수정 포인트는 네 군데입니다.

`Channel`, `Message`의 필수 컬럼명이 실제 모델과 다를 수 있습니다.
`JobType`, `JobTargetType`, `JobStatus`가 enum인지 문자열인지에 따라 비교식을 조금 바꿔야 할 수 있습니다.
프로젝트에 이미 `conftest.py` 공용 DB fixture가 있으면 이 파일의 `engine/session_factory/async_session` fixture는 제거하고 공용 fixture를 쓰는 편이 좋습니다.
`Base` import 경로와 `AsyncSessionLocal` 구조가 실제 프로젝트와 다르면 import를 맞춰야 합니다.

---

### 이 테스트 파일이 검증하는 핵심

이 파일이 통과하면 최소한 아래는 확인됩니다.

* `PreprocessWorker`가 idle일 때 `False`를 반환한다
* `normalize` job을 소비해 `NormalizedMessage`를 만든다
* normalize 결과로 `extract_features` job을 만든다
* `extract_features` job을 소비해 `MessageFeature`를 만든다
* feature 결과로 `single_message_summary` job을 만든다
* 입력이 없을 때 job이 `FAILED/DEAD_LETTER`로 정상 전이된다

즉, 지금 설계하신 **서비스 레이어 도입 + 전처리 워커 추가 + 추론 워커로의 연결**이 실제로 동작하는지 검증하는 최소 세트입니다.
