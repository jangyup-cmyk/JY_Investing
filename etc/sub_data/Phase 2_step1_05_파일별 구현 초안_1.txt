# 4순위: Worker 래퍼 함수 준비

이건 반드시 지금 해두는 게 좋습니다.

핵심은 “실시간 이벤트”와 “백필 메시지”가 **동일한 저장 진입점**을 쓰게 만드는 것입니다.

## 추천 최소 구성

### `collector_worker.py`

지금 단계에서는 아래 3개만 있어도 충분합니다.

* `persist_message_from_event(...)`
* `persist_message_from_history(...)`
* `safe_process_message(...)`

여기서 중요한 점은
`persist_message_from_event()`와 `persist_message_from_history()`가 내부적으로 결국 같은 저장 경로를 타야 한다는 것입니다.

예를 들면 구조는 이런 식이면 됩니다.

```python
async def persist_message_payload(db, payload):
    result = await upsert_message(db, payload)
    if result.action in {CREATED, UPDATED, UNCHANGED}:
        await update_channel_checkpoint(...)
    return result
```

그 위에

* 이벤트용: Telethon event/message → payload 변환 후 호출
* 백필용: history message → payload 변환 후 호출

이렇게 얹는 방식입니다.

즉 **진짜 공통 함수는 payload 기반 저장 래퍼**여야 합니다.

---

### `startup_tasks.py`

여기서는 지금 당장 백필 본문까지 짤 필요 없습니다.
대신 아래 시그니처는 잡아두는 게 좋습니다.

* `find_channels_pending_backfill()`
* `run_startup_recovery()`
* `mark_backfill_completed()`
* `mark_backfill_failed()`

이렇게 해두면 STEP 2에서 실제 루프만 채우면 됩니다.

---

### `client.py`

여기도 지금은 선언 위주로 충분합니다.

* `register_new_message_handler(handler)`
* `register_edited_message_handler(handler)`

핵심은 나중에 worker가 이 핸들러에 무엇을 넘겨받는지 인터페이스가 고정되는 것입니다.
