테스트 구조 정리

Telegram 수집 구조 개선
설정 기준 정리
문서는 TELEGRAM_MONITOR_FOLDERS, TELEGRAM_CHANNEL_GROUPS 기준으로 맞췄지만, 테스트 쪽에는 예전 TELEGRAM_FOLDER_CONFIGS 참조가 남아 있습니다. 먼저 이 불일치를 제거해야 합니다.

수집 부작용 분리
메시지 조회 로직과 파일 저장 로직을 명확히 분리하면, 실제 Telegram 연결 없이 저장 로직을 테스트할 수 있습니다.

중복 수집 상태 관리 개선
현재 last_message_ids가 메모리 안에만 있어 재시작하면 최근 메시지를 다시 저장할 수 있습니다.
최소 개선은 “실행 중 중복 방지”를 유지하고, 향후에는 상태 파일 저장 여부를 검토합니다.

통합 테스트 안전화
pytest -m integration 실행 시 실제 세션 파일을 건드린다는 점을 더 명확히 하고, 가능하면 테스트 전용 세션 파일 경로를 쓰도록 바꿀 수 있습니다.

검증
기본 테스트는 계속 외부 API 없이 통과해야 합니다.
Telegram 실제 연결 테스트는 명시적으로 pytest -m integration일 때만 실행됩니다.
