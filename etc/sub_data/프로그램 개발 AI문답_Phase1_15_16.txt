# 8) 각 파일의 “완료 기준” 한 줄 요약

| 파일                    | 완료 기준                               |
| --------------------- | ----------------------------------- |
| `requirements.txt`    | pip install 오류 없이 설치 가능             |
| `.env.example`        | APP/DB/LOG/TELEGRAM/COLLECTOR 구조 포함 |
| `settings.py`         | `.env` 값이 settings 객체로 로드됨          |
| `logger.py`           | 콘솔/파일 로그 둘 다 출력됨                    |
| `exceptions.py`       | 공통 예외 5종 정의됨                        |
| `responses.py`        | 성공/실패 JSON helper 사용 가능             |
| `base.py`             | ORM Base 선언 완료                      |
| `session.py`          | DB engine/session/check 함수 동작       |
| `models.py`           | 최소 3개 모델 import 가능                  |
| `app.py`              | FastAPI 앱 생성 및 라우터 등록 가능            |
| `health.py`           | `/health`, `/ready` 응답 가능           |
| `run_web.py`          | 웹 서버 실행 가능                          |
| `startup_tasks.py`    | worker 시작 태스크 함수 호출 가능              |
| `collector_worker.py` | worker 시작/DB 확인/로그 출력 가능            |
| `run_collector.py`    | 워커 독립 실행 가능                         |

---

# 9) 가장 실무적인 작업 방식

Cursor에서 작업할 때는 아래처럼 하면 좋습니다.

### 1차

웹 서버 뜨는 데 필요한 파일만 먼저 생성

### 2차

DB/예외/응답 구조 정리

### 3차

워커 실행 뼈대 생성

즉, 처음부터 전부 한 번에 하려고 하지 말고
**웹 먼저 → 공통 정리 → 워커 추가** 순서가 가장 안전합니다.

---

# 10) 최종 요약

지금 각 파일에 먼저 넣어야 할 최소 내용은:

* **설정은 단순하지만 통합적으로**
* **로그는 콘솔+파일**
* **DB는 최소 모델만**
* **웹은 health/ready까지만**
* **워커는 실행 뼈대까지만**
* **텔레그램 상세 수집은 아직 금지**

이 기준이면 Phase 1이 과하지 않으면서도, Phase 2로 바로 넘어가기 좋은 상태가 됩니다.



원하시면 다음으로
**각 프롬프트를 더 짧고 강하게 다듬은 “붙여넣기 최적화 버전”**으로 다시 압축해드리겠습니다.
