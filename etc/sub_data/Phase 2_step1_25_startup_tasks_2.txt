## 2) `backfill_missed_messages()` 교체

이 함수는 `BackfillDeferredError`를 일반 예외보다 먼저 잡아야 합니다.

```python
async def backfill_missed_messages(collector=None, persist_func=None) -> None:
    """
    [백필(Backfill) 핵심 원칙]
    - 대상: 수집 활성화 상태이면서 PENDING_BACKFILL 상태인 채널
    - 범위: 최근 3일 또는 최근 1,000건
    - 동작: 백필 완료 후 ACTIVE 전이
    """
    logger.info("PENDING_BACKFILL 상태의 채널 스캔 시작...")

    async with get_session() as db:
        pending_channels = await find_channels_pending_backfill(db)

        if not pending_channels:
            logger.info("현재 백필이 필요한 채널이 없습니다.")
            return

        logger.info(f"총 {len(pending_channels)}개의 채널이 백필 대기 중입니다.")

        if collector is None or persist_func is None:
            logger.info("collector/persist_func 미주입 상태이므로 실제 백필은 건너뜁니다.")
            return

        for channel, state in pending_channels:
            logger.info(f"백필 대상 채널: [{channel.title}] ({channel.tg_channel_id})")

            try:
                processed_count = await perform_backfill_for_channel(
                    db=db,
                    collector=collector,
                    channel=channel,
                    persist_func=persist_func,
                    days=BACKFILL_DAYS,
                    limit=BACKFILL_LIMIT,
                )

                await mark_backfill_completed(db, channel.id)
                await db.commit()

                logger.info(
                    f"백필 완료 처리: [{channel.title}] ({channel.tg_channel_id}) "
                    f"messages={processed_count}"
                )

            except BackfillDeferredError as e:
                await db.rollback()

                # FloodWait 장기 대기 등은 장애가 아니라 "연기"로 취급
                await mark_backfill_deferred(db, channel.id, str(e))
                await db.commit()

                logger.warning(
                    f"백필 연기 처리: [{channel.title}] ({channel.tg_channel_id}) {e}"
                )

            except Exception as e:
                await db.rollback()

                await mark_backfill_failed(db, channel.id, str(e))
                await db.commit()

                logger.error(
                    f"백필 실패 처리: [{channel.title}] ({channel.tg_channel_id}) error={e}"
                )
```

## 3) `mark_backfill_deferred()` 추가

이 함수는 상태를 **`ERROR`로 바꾸지 않고 `PENDING_BACKFILL` 유지**합니다.

```python
async def mark_backfill_deferred(db, channel_id: int, reason: str) -> None:
    """
    FloodWait 장기 대기 등으로 이번 백필 사이클을 연기할 때 사용.
    상태는 PENDING_BACKFILL 유지, last_error_message만 기록.
    """
    stmt = select(TelegramChannelCollectionState).where(
        TelegramChannelCollectionState.telegram_channel_id == channel_id
    )
    result = await db.execute(stmt)
    state = result.scalar_one_or_none()

    now = datetime.now(timezone.utc)

    if state is None:
        state = TelegramChannelCollectionState(
            telegram_channel_id=channel_id,
            status=ChannelCollectionStatus.PENDING_BACKFILL.value,
            last_error_message=reason,
            last_error_at=now,
        )
        db.add(state)
    else:
        state.status = ChannelCollectionStatus.PENDING_BACKFILL.value
        state.last_error_message = reason
        state.last_error_at = now
```

## 4) `mark_backfill_completed()`는 그대로 두되 clear 정책만 유지

```python
async def mark_backfill_completed(db, channel_id: int) -> None:
    """백필 완료 시 ACTIVE로 전환하고 에러 상태를 정리합니다."""
    stmt = select(TelegramChannelCollectionState).where(
        TelegramChannelCollectionState.telegram_channel_id == channel_id
    )
    result = await db.execute(stmt)
    state = result.scalar_one_or_none()

    now = datetime.now(timezone.utc)

    if state is None:
        state = TelegramChannelCollectionState(
            telegram_channel_id=channel_id,
            status=ChannelCollectionStatus.ACTIVE.value,
            last_backfill_at=now,
            last_error_message=None,
            last_error_at=None,
        )
        db.add(state)
    else:
        state.status = ChannelCollectionStatus.ACTIVE.value
        state.last_backfill_at = now
        state.last_error_message = None
        state.last_error_at = None
```

## 5) `mark_backfill_failed()`는 진짜 실패만 ERROR

```python
async def mark_backfill_failed(db, channel_id: int, error_message: str) -> None:
    """백필 실패 시 ERROR로 전환하고 에러 메시지를 기록합니다."""
    stmt = select(TelegramChannelCollectionState).where(
        TelegramChannelCollectionState.telegram_channel_id == channel_id
    )
    result = await db.execute(stmt)
    state = result.scalar_one_or_none()

    now = datetime.now(timezone.utc)

    if state is None:
        state = TelegramChannelCollectionState(
            telegram_channel_id=channel_id,
            status=ChannelCollectionStatus.ERROR.value,
            last_error_message=error_message,
            last_error_at=now,
        )
        db.add(state)
    else:
        state.status = ChannelCollectionStatus.ERROR.value
        state.last_error_message = error_message
        state.last_error_at = now
```

## 의미 정리

이제 상태 전이는 이렇게 됩니다.

* **정상 완료** → `ACTIVE`
* **FloodWait 장기 대기** → `PENDING_BACKFILL 유지`
* **진짜 실패** → `ERROR`

이 구분이 들어가면 운영할 때 훨씬 명확합니다.

다음으로 가장 자연스러운 작업은 **`tests/test_startup_tasks.py`에 `mark_backfill_deferred()` 테스트까지 반영한 전체본 정리**입니다.


좋습니다.
아래는 **`mark_backfill_deferred()`까지 반영한 `tests/test_startup_tasks.py` 전체본**입니다.

주의할 점은 하나입니다.

* 상태 테이블 FK가 `telegram_channel_id`면 아래 그대로 사용
* 실제 모델이 `channel_id`면 파일 전체에서 그 부분만 바꾸면 됩니다

class FakeCollector:
    def __init__(self, messages):
        self.messages = messages
        self.called_with = None

    async def iter_channel_messages(self, entity, limit=1000, days=3):
        self.called_with = {
            "entity": entity,
            "limit": limit,
            "days": days,
        }
        for msg in self.messages:
            yield msg
