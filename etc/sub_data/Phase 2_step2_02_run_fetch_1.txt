원인은 명확합니다.

지금은 **코드(models.py / startup_tasks.py)는 새 컬럼을 쓰고 있는데**, 실제 DB 파일 `./data/telegram.db` 안의 `telegram_channel_collection_state` 테이블에는 그 컬럼이 아직 없습니다.

에러 핵심:

* 코드가 조회하려는 컬럼
  `backfill_next_offset_id`, `backfill_started_at`, `backfill_finished_at`
* 실제 SQLite 테이블
  아직 없음
* 그래서 발생한 오류
  `no such column: telegram_channel_collection_state.backfill_next_offset_id`

또 한 가지 중요한 점은 `init_db()`는 보통 **없는 테이블만 생성**하지, **기존 테이블 구조를 자동 변경하지는 않습니다.**
그래서 코드만 바꾸고 실행하면 지금처럼 바로 터집니다.

### 가장 빠른 해결법

개발 DB를 새로 만들어도 되는 상황이면, 기존 DB를 백업한 뒤 삭제하고 다시 생성하는 방법이 제일 간단합니다.

1. 실행 중지
2. `data/telegram.db` 백업
3. 기존 `data/telegram.db` 삭제
4. 다시 실행

이렇게 하면 새 모델 기준으로 테이블이 다시 만들어집니다.

다만 기존 수집 데이터가 중요하면 이 방법 말고 **마이그레이션**으로 가야 합니다.

### 데이터 유지가 필요할 때

SQLite에서 컬럼만 추가하면 됩니다.

아래 SQL을 실행하세요.

```sql
ALTER TABLE telegram_channel_collection_state
ADD COLUMN backfill_next_offset_id BIGINT;

ALTER TABLE telegram_channel_collection_state
ADD COLUMN backfill_started_at DATETIME;

ALTER TABLE telegram_channel_collection_state
ADD COLUMN backfill_finished_at DATETIME;
```

### SQLite에 바로 적용하는 방법

프로젝트 루트에서:

```bash
.venv\Scripts\python.exe
```

파이썬 콘솔에서:

```python
import sqlite3

conn = sqlite3.connect("data/telegram.db")
cur = conn.cursor()

print("컬럼 추가 완료")
```

### 이미 일부 컬럼이 추가된 상태일 수도 있으니 안전하게 하려면

아래처럼 하나씩 예외 무시로 넣는 편이 낫습니다.

```python
import sqlite3

conn = sqlite3.connect("data/telegram.db")
cur = conn.cursor()

queries = [
    "ALTER TABLE telegram_channel_collection_state ADD COLUMN backfill_next_offset_id BIGINT",
    "ALTER TABLE telegram_channel_collection_state ADD COLUMN backfill_started_at DATETIME",
    "ALTER TABLE telegram_channel_collection_state ADD COLUMN backfill_finished_at DATETIME",
]

for q in queries:
    try:
        cur.execute(q)
        print("OK:", q)
    except Exception as e:
        print("SKIP/ERR:", q, e)

conn.commit()
conn.close()
print("완료")
```

### 지금 로그에서 추가로 보이는 점

`Startup Tasks`는 실패했는데, 워커는 계속 실시간 루프로 들어갔습니다.
즉 지금 상태는:

* **실시간 수집 루프는 살아 있을 수 있음**
* **백필 기능만 죽어 있음**

그래서 스키마 수정 후에는 다시 실행해서 **startup backfill이 정상 통과하는지**까지 꼭 봐야 합니다.

가장 추천하는 순서는 이겁니다.

1. `data/telegram.db` 백업
2. 위 3개 컬럼 추가
3. 재실행
4. 다음 오류가 나오면 그 에러 기준으로 추가 수정

원하시면 바로 다음으로 **윈도우 CMD에서 한 번에 실행 가능한 컬럼 추가용 파이썬 스크립트** 형태로 정리해드리겠습니다.

아래 순서대로 하시면 됩니다.

먼저 **DB 백업**부터 하세요.

```bash
copy data\telegram.db data\telegram_backup_20260321.db
```

그 다음, 프로젝트 루트에 `add_backfill_columns.py` 파일을 하나 만드세요. 내용은 아래 그대로 넣으면 됩니다.

```python
import sqlite3

DB_PATH = "data/telegram.db"

queries = [
    "ALTER TABLE telegram_channel_collection_state ADD COLUMN backfill_next_offset_id BIGINT",
    "ALTER TABLE telegram_channel_collection_state ADD COLUMN backfill_started_at DATETIME",
    "ALTER TABLE telegram_channel_collection_state ADD COLUMN backfill_finished_at DATETIME",
]

conn = sqlite3.connect(DB_PATH)
cur = conn.cursor()

for q in queries:
    try:
        cur.execute(q)
        print(f"OK: {q}")
    except Exception as e:
        print(f"SKIP/ERR: {q} -> {e}")

conn.commit()
conn.close()

print("컬럼 추가 작업 완료")
```

저장 후 CMD에서 실행하세요.

```bash
.venv\Scripts\python.exe add_backfill_columns.py
```

정상이라면 대략 이런 식으로 나옵니다.

```text
OK: ALTER TABLE telegram_channel_collection_state ADD COLUMN backfill_next_offset_id BIGINT
OK: ALTER TABLE telegram_channel_collection_state ADD COLUMN backfill_started_at DATETIME
OK: ALTER TABLE telegram_channel_collection_state ADD COLUMN backfill_finished_at DATETIME
컬럼 추가 작업 완료
```

이미 한 번 추가된 컬럼이 있으면 이런 식으로 나올 수 있는데, 그건 괜찮습니다.

```text
SKIP/ERR: ... -> duplicate column name: backfill_next_offset_id
```

그 다음 다시 실행하세요.

```bash
.venv\Scripts\python.exe -m src.run_fetch
```

이후 확인 포인트는 이것입니다.

* `no such column: telegram_channel_collection_state.backfill_next_offset_id` 에러가 사라지는지
* `Startup Tasks 실행 중...` 다음에 백필 스캔이 정상 진행되는지
* 새로운 다른 에러가 나오면 그 에러가 다음 수정 대상입니다
